02 Manipulate HTML tags from R with {htmltools}

Published

January 28, 2026

Modified

January 28, 2026

欢迎大家关注我的公众号：RSSPhto

htmltools包脱胎自shiny，功能主要涉及两方面：创建HTML-tag和处理web依赖。它的最终目标是实现操纵、组合和重新排列标签，以便从R中创建灵活丰富的HTML结构。

安装htmltools包

# CRAN
install.packages("htmltools")
# development version
remotes::install_github("rstudio/htmltools")

下面是一个依赖htmltools包的例子

remotes::install_github("RinteRface/shinyRPG")
library(shinyRPG)
shinyRPGDemo()

Writing HTML Tags from R

使用htmltools包可以很方便地创建HTML标签。这类创建标签的函数有一个共同点：有name的参数会被转换为标签的属性（attributes），没有name的参数被转换为标签包裹的内容（children）。

library(htmltools)
div(id = "header", class = "main", "Hello", strong("World"))
Hello World

某些标签具有空属性（empty attributes），比如<input disabled>标签，此时可以用input(disabled = NA)表示。

tags$input(disabled = NA)

Notations

在R终端中使用htmltools::tags$时可以显示可用的标签，同时为了方便，有些标签拥有简单的函数，例如div()，h1()等。

# good
h1("This is a title")

This is a title


# correct but not necessary
tags$h1("This is a title")

This is a title

当然有些标签只能通过tags$来创建，比如上面的<input>标签。

input(disabled = NA)
#> Error in `input()`:
#> ! could not find function "input"
tags$input(disabled = NA)

当需要使用多个tags$<TAG_NAME>函数时，可以使用withTags()函数，来避免重复输入tags$：

# Better
withTags(
  nav(div(), ul(li(), li()))
)

# instead of
tags$nav(div(), tags$ul(tags$li(), tags$li()))

当你想将多个标签收集到一起时，是用tagList()，而不是list()。前者结果中拥有额外的class属性——shiny.tag.list，导致结果完全不同：

# good
tag_list_1 <- tagList(
  p("Some text"),
  div("Content")
)

# correct but not optimal
tag_list_2 <- list(
  p("Some text"),
  div("Content")
)

Adding new tags

使用tag()可以自定义HTML标签，但是要遵循HTML的命名规则。

customTag <- tag(
  "test",
  list(class = "test", p("Custom Tag"))
)
str(customTag)
#> List of 3
#>  $ name    : chr "test"
#>  $ attribs :List of 1
#>   ..$ class: chr "test"
#>  $ children:List of 1
#>   ..$ :List of 3
#>   .. ..$ name    : chr "p"
#>   .. ..$ attribs : Named list()
#>   .. ..$ children:List of 1
#>   .. .. ..$ : chr "Custom Tag"
#>   .. ..- attr(*, "class")= chr "shiny.tag"
#>  - attr(*, "class")= chr "shiny.tag"

Alternative way to write tags

HTML()函数可以直接包裹HTML文本信息，避免发生转义：

div(HTML("Hello <u>World</u>"))
Hello World 
div("Hello <u>World</u>")
Hello <u>World</u>

下面代码的结果相同，但是它们的class属性不同：

HTML("<div>Blabla</div>")
Blabla
div("Blabla")
Blabla

class(HTML("<div>Blabla</div>"))
#> [1] "html"      "character"
class(div("Blabla"))
#> [1] "shiny.tag"

如果你想转换HTML文本为R code，可以使用html2R包，但无法处理非标准属性。
